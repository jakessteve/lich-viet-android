## Format Rules

1. **Language:** Findings and user quotes MUST be in Vietnamese user language (non-technical). Technical analysis goes in parenthetical English notes if needed.
2. **Scores:** Always integers 1–5. No half scores. Average to one decimal.
3. **Screenshots:** MANDATORY for every page tested. At minimum one per page.
4. **Severity:** Every issue MUST have a severity icon (//).
5. **Trend:** If a previous report exists, the Trend Comparison section is MANDATORY.
6. **Persona voice:** Each finding should indicate which persona experienced it.
7. **Actionable:** Every issue MUST have a corresponding suggestion.
8. **File path:** `.hc/feedback/user-feedback-YYYY-MM-DD.md` — no exceptions.
9. **Screenshots path:** `.hc/feedback/screenshots/` — organized by date.

---

## Consumption Contract

Other agents MUST be able to parse this report:
- **@pm:** Reads "Priority Recommendations" and "Executive Summary" for backlog
- **@designer:** Reads "UI/UX" dimension scores and "Detailed Findings" for design fixes
- **@dev-fe/@dev-be:** Reads "Speed" scores and "Task Completion Tests" for performance work
- **@qc:** Reads "Task Completion Tests" to add user-centric test cases
